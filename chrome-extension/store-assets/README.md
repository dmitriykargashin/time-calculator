# Chrome Web Store listing images

Generated by [`build.mjs`](build.mjs) — the real panel (`src/popup/panel.html`) is
rendered in an iframe at a true side-panel width (light/dark, with Convert/Rate
opened), then composed onto branded canvases. Everything is captured at 2× with
headless Chrome and downscaled + flattened to **opaque 24-bit RGB (no alpha)** —
what the store requires.

Regenerate (from `chrome-extension/`, after `npm run build` so `dist/` exists):

```
node store-assets/build.mjs
```

## What to upload where

| File | Size | Chrome Web Store slot | Notes |
|---|---|---|---|
| `screenshot-1.png` … `screenshot-5.png` | 1280×800 | **Screenshots** (up to 5) | 1 required, 5 is the max. |
| `promo-small-440x280.png` | 440×280 | **Small promo tile** | The storefront thumbnail — leads with the real calculator UI (card + colored expression + green result + action row). |
| `promo-marquee-1400x560.png` | 1400×560 | **Marquee promo tile** | The banner used when the store **features** the item. |
| `store-icon-128.png` | 128×128 | **Store icon** | Same as the manifest 128 icon; the store also pulls this from the manifest automatically. |

Screenshots in order: **side-panel hero** (a Chrome window with the calculator
docked on the right, so "it's a side panel" reads instantly) · Convert (all units) ·
Rate (duration → rate) · **History** (seeded, real entries) · Omnibox (`tc` in the
address bar). Warm editorial style — cream + Cardamon green, quiet atmospheric
background (soft gradient, faint streaks, sparse low-opacity accents kept off the
text), heavy two-tone Hanken headlines, and the panel rendered WIDE so the action
row never wraps. Dark theme isn't a dedicated screenshot right now (5-slot limit);
say the word and I'll swap one in or add a dark peek.

Specs (Chrome Web Store, current): screenshots 1280×800 **or** 640×400, 24-bit PNG
(no alpha) or JPEG — 1280×800 recommended; small tile 440×280; marquee 1400×560;
PNG/JPEG, no alpha. `.build/` is a scratch dir (gitignored).

## Featuring

The blue **Featured** badge is curated by the Chrome Web Store team — there is no
paid slot or apply button. It goes to items that meet their quality bar: Manifest
V3, least-privilege permissions, a complete high-quality listing (these images), a
clear description + category, and established trust (verified publisher, a privacy
policy, accurate data-use disclosures). Uploading the small + marquee promo tiles
makes the item eligible to be surfaced; the rest raises the odds.
